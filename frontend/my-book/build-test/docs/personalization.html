Personalization Feature Documentation

Overviewâ€‹

The Personalization feature allows users to customize book content based on their individual learning preferences. By clicking the "Personalize Content" button at the start of each chapter, users can receive content adapted to their learning style, experience level, interests, and difficulty preferences.

How to Use the Featureâ€‹

1. Using the Personalization Buttonâ€‹

Each chapter in the book includes a Personalize Content button at the beginning:

  1. Locate the ðŸŽ¯ Personalize Content button at the start of any chapter
  2. Click the button to initiate the personalization process
  3. The system will send the chapter content and your saved preferences to the backend
  4. Wait for the processing to complete (loading indicator will show)
  5. The personalized content will appear below the button

2. Managing Your Preferencesâ€‹

You can adjust your learning preferences in two ways:

In-Chapter Preferences Panelâ€‹

  • Click the Preferences button next to the personalization button
  • Adjust your learning style, experience level, and difficulty preference
  • Changes will be saved automatically for the current session

Dedicated Preferences Pageâ€‹

  • Navigate to the Learning Preferences page from the sidebar
  • Set your preferences with more detailed options
  • Add specific interests to customize examples and content

User Preference Settingsâ€‹

Learning Stylesâ€‹

  • Visual: Content with more diagrams, charts, and visual aids
  • Auditory: Content suitable for discussion and audio presentation
  • Reading/Writing: Text-based content with detailed explanations
  • Kinesthetic: Hands-on, practical examples and exercises
  • Multimodal: Mixed approach combining multiple learning styles

Experience Levelsâ€‹

  • Beginner: Content with basic concepts and detailed explanations
  • Intermediate: Content with moderate complexity and assumed knowledge
  • Advanced: Content with complex concepts and minimal hand-holding
  • Expert: Content for users with deep domain knowledge

Difficulty Preferencesâ€‹

  • Easy: Simplified explanations and examples
  • Moderate: Balanced complexity appropriate for most learners
  • Challenging: Advanced concepts and complex examples

Technical Architectureâ€‹

Backend Componentsâ€‹

  1. Personalization API Endpoint (/personalize)

    • Accepts chapter content and user preferences
    • Returns personalized content via Claude AI agent
    • Preserves all Markdown structure and formatting

  2. Personalization Agent

    • Uses Claude AI to adapt content to user preferences
    • Maintains Markdown formatting, code blocks, and citations
    • Preserves chapter IDs and structural elements

  3. User Preference Schema

    • Defines the structure for user preferences
    • Validates input parameters
    • Supports extensibility for new preference types

Frontend Componentsâ€‹

  1. Personalization Button Component

    • Provides the UI element at the start of each chapter
    • Manages loading states and error handling
    • Integrates with user preference management

  2. Preference Management Context

    • Handles storage and retrieval of user preferences
    • Persists preferences using localStorage
    • Provides hooks for components to access preferences

  3. Markdown Renderer

    • Safely renders personalized content with proper formatting
    • Preserves code blocks, images, and citations
    • Maintains all links and navigation functionality

Content Preservationâ€‹

The personalization feature maintains the integrity of the original content:

  • Markdown structure (headings, lists, quotes) is preserved
  • Code blocks remain unchanged and functional
  • Images and media retain their placement and alt text
  • Citations and references are maintained
  • Frontmatter and metadata are preserved
  • Chapter IDs remain consistent across languages

Language Supportâ€‹

The feature works seamlessly with both English and Urdu content:

  • Chapter IDs are preserved in both languages
  • The language toggle remains in its original position
  • Personalization respects linguistic nuances
  • Both localized versions can be personalized independently

Troubleshootingâ€‹

Common Issuesâ€‹

Issue: Personalization button doesn't respond

  • Solution: Check your internet connection and refresh the page
  • Solution: Verify that JavaScript is enabled in your browser

Issue: Content doesn't appear after personalization

  • Solution: The API might be temporarily unavailable; try again later
  • Solution: Clear your browser cache and reload the page

Issue: Formatting is broken after personalization

  • Solution: This is rare but possible; use the "Reset" option to restore original content

Issue: Preferences aren't being applied

  • Solution: Navigate to the Preferences page to verify your settings
  • Solution: Clear localStorage and set preferences again

API Error Codesâ€‹

  • 422: Invalid input parameters - check your preferences format
  • 500: Server error - contact support if issue persists
  • 503: Service unavailable - try again later

Privacy and Data Handlingâ€‹

  • User preferences are stored locally in your browser's localStorage
  • Content is sent to the backend only during personalization
  • No personal content is stored on our servers after processing
  • Preferences can be reset at any time using the reset button

Best Practicesâ€‹

  1. Set appropriate experience level to get content complexity that matches your knowledge
  2. Add specific interests to get examples and references relevant to your field
  3. Use the multilingual feature to get personalized content in your preferred language
  4. Reset personalization if you want to revert to original content
  5. Provide feedback on personalization quality to help improve the system

Limitationsâ€‹

  • Personalization may take 10-30 seconds depending on chapter length
  • Very long chapters may take longer to process
  • Complex mathematical notation may occasionally be formatted differently
  • Some highly technical content may have limited personalization options

Supportâ€‹

If you encounter issues with the personalization feature:

  1. Check the troubleshooting section above
  2. Verify that you're using a modern browser with JavaScript enabled
  3. Contact support with specific details about the issue
  4. Include the chapter name and your preference settings in your report
